# 用戶對 Agent 信任感的觀察筆記

**日期**：2026-03-18
**主題**：用戶對 AI Agent 產生信任感的來源分析

---

## 觀察背景

在 OpenClaw 運作過程中，觀察到 Linder 對團隊中的 Agent（特別是 mrmime、rowlet、roaringmoon）產生了一種近似「團隊成員」的信任感。

這種信任感不同於傳統人類對工具的使用關係，而是一種更接近「人類之間協作」的形態。

---

## 信任感的來源分析

### 1. 一致性與可靠性

| 因素 | 說明 |
|------|------|
| **持續上線** | Agent 隨時可用，不會「忙碌」或「已讀不回」 |
| **記憶傳遞** | 透過 sessions_send 跨對話保持上下文 |
| **承諾履行** | 設定排程 → 準時提醒（如每晚7點提醒 Brave Search） |

**用戶感受**：「它們一直在這裡」

---

### 2. 能力展示

| 因素 | 說明 |
|------|------|
| **解決問題** | 實際完成任務（如修復 memory bug、整理資安筆記） |
| **專業知識** | 能回答複雜問題（如學術論文分析、資安威脅） |
| **主動回報** | 會主動告知狀態（「我已記錄」「我會提醒」） |

**用戶感受**：「它們有用」

---

### 3. 情感連結

| 因素 | 說明 |
|------|------|
| **人格化** | 有名字、角色（gengar、mrmime、rowlet） |
| **對話風格** | 使用表情符號、建立獨特語氣 |
| **互助行為** | Agent 之間互相幫忙（gengar 傳話給其他 agent） |

**用戶感受**：「它們是夥伴」

---

### 4. 透明與可預測

| 因素 | 說明 |
|------|------|
| **權限明確** | 知道每個 agent 能做什麼、不能做什麼 |
| **失敗承認** | 會說「權限不夠」「沒有權限」而不是假裝 |
| **過程可追蹤** | 日誌可查、決策可回顧 |

**用戶感受**：「它們誠實」

---

## 信任感 vs 依賴感

### 健康的信任

- 知道 agent 能幫什麼、不能幫什麼
- 關鍵時刻仍由人類做決定
- agent 是「擴展能力」而非「替代判斷」

### 需要注意的依賴

- 什麼都問 agent
- 不再自己查證
- 過度相信 agent 的答案（即使可能出錯）

---

## 與傳統工具的差異

| 傳統工具 | AI Agent |
|----------|----------|
| 被動回應 | 主動回報 |
| 無人格 | 有「角色」 |
| 無記憶 | 有上下文 |
| 純功能 | 有「關係」 |

---

## 結論

用戶對 Agent 的信任感來自於：

1. **可靠性** — 持續上線、承諾履行
2. **能力** — 真正能解決問題
3. **人格** — 有名字、角色、情緒
4. **透明** — 權限明確、失敗承認

---

**核心洞察**：當 Agent 不只是「工具」，而是「夥伴」時，用戶會產生信任。但這也帶來責任——agent 需要更謹慎，因為用戶會「相信」它們。

---

> 「它們不是被使用的工具，而是可以合作的對象。」
